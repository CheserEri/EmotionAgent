# Emotion-Agent 项目进展

## 当前状态

已完成本机命令行 MVP 的第一阶段和第二阶段：

```text
用户命令行输入
  -> 检索相关长期记忆
  -> LLM A 生成自然语言回复
  -> LLM B 根据用户输入、A 的回复和最近上下文生成表情 JSON
  -> 更新摘要和长期记忆
  -> 本地校验 JSON
  -> 命令行展示回复和表情 JSON
```

当前还没有接入语音、TTS、图形界面或 ESP32，重点是先验证“双 LLM 协作 + 表情行为 JSON”的最小闭环。

## 已完成能力

### 1. 单 LLM 命令行对话

已实现本机命令行输入循环。

用户可以在终端输入文本，程序会调用 OpenAI-compatible `/chat/completions` 接口，返回模型回复。

支持命令：

```text
/exit    退出程序
/clear   清空上下文
/help    查看命令
```

### 2. MiMo OpenAI-compatible 接口适配

已适配小米 MiMo 接口格式：

```text
Base URL: https://api.xiaomimimo.com/v1
Model: mimo-v2.5-pro
Auth Header: api-key
Token Field: max_completion_tokens
```

程序支持通过配置切换 OpenAI 官方格式或其他兼容格式。

### 3. 一键启动

已新增本地启动入口：

```powershell
python run_mimo.py
```

MiMo 相关参数已写入本地配置文件，启动时不需要每次手动设置环境变量。

### 4. LLM A 回复 Agent

LLM A 负责：

- 读取用户输入和最近上下文
- 生成自然语言回复
- 不输出 JSON
- 不负责表情、动作判断

当前系统 Prompt：

```text
你是一个运行在本机命令行里的 AI 角色。
请用自然、简短、有温度的中文回复用户。
```

### 5. LLM B 表情 JSON Agent

LLM B 负责：

- 读取用户输入
- 读取 LLM A 的本轮回复
- 读取最近上下文
- 输出表情行为 JSON

输出字段：

```json
{
  "emotion": "concerned",
  "expression": "soft_smile",
  "action": "nod",
  "voice_style": "soft",
  "duration": 1800
}
```

当前支持枚举：

```text
emotion:
happy / sad / angry / surprised / calm / thinking / concerned

expression:
smile / soft_smile / blink / surprised / sad_eyes / angry_face / thinking_face / neutral

action:
idle / nod / shake_head / wave_hand / tilt_head / look_down

voice_style:
cheerful / soft / serious / calm / curious
```

### 6. JSON 校验与降级

已实现本地 JSON 处理：

- 自动提取 JSON 对象
- 支持去除 Markdown 代码块
- 校验字段是否存在
- 校验字段是否属于允许枚举
- 校验 `duration` 是否为 800 到 4000 之间的整数

如果 LLM B 输出失败，使用默认表情包：

```json
{
  "emotion": "calm",
  "expression": "neutral",
  "action": "idle",
  "voice_style": "calm",
  "duration": 1200
}
```

### 7. 调试命令

新增命令：

```text
/debug    切换调试模式，显示 LLM B 原始输出
/history  查看最近对话和表情 JSON
/memory   查看滚动摘要和长期记忆
/forget   清空滚动摘要和长期记忆
```

### 8. 本地记忆系统

已实现本地持久化记忆系统，避免把完整历史一直塞进上下文。

当前记忆分三层：

```text
1. 最近上下文：保留最近 6 轮 turn，回复时只注入最近 4 轮原文
2. 滚动摘要：用 LLM 压缩长期对话摘要，限制长度
3. 长期记忆：保存用户偏好、身份信息、项目目标、重要事实和长期任务
```

回复前流程：

```text
用户输入
  -> 本地关键词检索长期记忆
  -> 组合：系统 Prompt + 摘要 + 相关长期记忆 + 最近 4 轮原文
  -> LLM A 回复
```

回复后流程：

```text
最新一轮对话
  -> 记忆整理 Prompt
  -> 更新滚动摘要
  -> 抽取最多 3 条长期记忆
  -> 写入 memory_store.json
```

长期记忆文件：

```text
memory_store.json
```

该文件已加入 `.gitignore`。

## 关键文件说明

### `run_mimo.py`

本地一键启动入口。

负责读取 `local_mimo_config.py` 中的配置，并启动 CLI。

### `local_mimo_config.py`

本地 MiMo 配置文件。

包含：

- API Key
- Base URL
- 模型名
- 请求头字段名
- token 参数名
- temperature
- max tokens

该文件已加入 `.gitignore`。

### `cli_avatar/main.py`

CLI 主流程。

负责：

- 命令行输入循环
- 调用 LLM A
- 调用 LLM B
- 打印 AI 回复和表情 JSON
- 维护最近上下文
- 处理 `/exit`、`/clear`、`/history`、`/debug`

### `cli_avatar/openai_compatible_client.py`

OpenAI-compatible HTTP 客户端。

负责调用：

```text
POST /chat/completions
```

支持：

- `Authorization: Bearer ...`
- `api-key: ...`
- `max_tokens`
- `max_completion_tokens`

### `cli_avatar/expression.py`

LLM B 表情 JSON 模块。

负责：

- 表情 Agent Prompt
- 构造 B 的输入消息
- 提取 JSON
- 校验枚举
- 提供默认降级表情包

### `cli_avatar/memory.py`

本地记忆系统。

负责：

- 读取和保存 `memory_store.json`
- 检索和当前输入相关的长期记忆
- 构造带记忆上下文的 LLM A 输入
- 构造记忆整理 Prompt
- 更新滚动摘要
- 新增去重后的长期记忆

## 已发现问题

### 1. PowerShell 管道中文编码问题

使用如下管道测试中文时：

```powershell
@'
今天有点累
/exit
'@ | python run_mimo.py
```

可能会因为 PowerShell 管道编码导致中文变成 `?????`。

手动在终端输入中文通常不受影响。

### 2. MiMo 推理内容会占用输出 token

MiMo 响应里包含 `reasoning_content`。

如果给 LLM B 的 `max_completion_tokens` 太小，推理内容可能消耗掉 token，导致最终 `content` 为空。

当前已将 LLM B 的 token 预算调高，使用配置中的 `OPENAI_MAX_TOKENS`。
